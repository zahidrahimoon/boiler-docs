# Layout 

This guide explains the **dynamic layout system** in the project, including layout types, runtime behavior, and developer usage patterns.

## Configuration: `LAYOUT_CONFIGS`

```ts
export const LAYOUT_CONFIGS = {
  website: { showHeader: true, showSidebar: false, showFooter: true },
  dashboard: { showHeader: true, showSidebar: true, showFooter: false }
} as const;
```

## Developer Workflow

Sometimes, you want a **specific page to use a certain layout**. For instance, a `Home` page should use the `website` layout:

```tsx
"use client";
import { useEffect } from "react";
import { useLayout } from "@/contexts/layout-context";
import DynamicLayout from "@/components/layout/DynamicLayout";

export default function HomePage() {
  const { setLayoutType } = useLayout();

  useEffect(() => {
    setLayoutType("website");
  }, [setLayoutType]);

  return (
    <DynamicLayout>
      <h1>Welcome to the Home Page</h1>
      <p>This page uses the website layout with header + footer.</p>
    </DynamicLayout>
  );
}
```

âš¡ Wrapping the page in `<DynamicLayout>` ensures that all layout-related configs, sidebars, and headers are correctly applied.
</Callout>
