# Components Overview

This document provides a structured overview of how components are organized in the project.  
The system is divided into four main categories:

- **shared** — cross-module reusable building blocks  
- **ui (shadcn)** — design-system primitives and composite UI controls  
- **layout** — structural and navigation components  
- **form** — form primitives, field types, and validation wiring  

Each category below includes a concise explanation and a **table containing the exact components within that category**.

---

# Developer Notes

<Callout type="note">
Use these categories when deciding where a new component belongs. This ensures the architecture stays scalable and consistent.
</Callout>

- Add reusable visual helpers to **shared**  
- Add design primitives or composite UI controls to **ui**  
- Add navigation & structural components to **layout**  
- Add any form input, schema, or wrapper to **form**
