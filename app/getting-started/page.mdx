# Getting Started

Get your Next.js Enterprise Starter Kit up and running in minutes with this comprehensive setup guide.

## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js 18+** - [Download from nodejs.org](https://nodejs.org/)
- **pnpm** (recommended) or **npm** - Package manager
- **Git** - Version control system

### Verify Installation

```bash
node --version  # Should be 18.0.0 or higher
npm --version   # Should be 8.0.0 or higher
git --version   # Any recent version
```

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/zahidrahimoon/next-starter.git
cd nextjs-starter
```

### 2. Install Dependencies

Using npm (recommended):
```bash
npm install
```

Using pnpm:
```bash
pnpm install
```

Using yarn:
```bash
yarn install
```

### 3. Environment Setup

Copy the example environment file and configure your variables:

```bash
cp .env.example .env.local
```

Edit `.env.local` with your configuration:

```bash
# API Configuration
NEXT_PUBLIC_API_URL=http://localhost:3001/api

# Authentication
NEXTAUTH_SECRET=your-secret-key-here
NEXTAUTH_URL=http://localhost:3000

# Database (if using)
DATABASE_URL=postgresql://username:password@localhost:5432/dbname

# Socket.IO (for real-time features)
NEXT_PUBLIC_SOCKET_URL=http://localhost:4000

# Optional: External Services
NEXT_PUBLIC_ANALYTICS_ID=your-analytics-id
```

### 4. Run the Development Server

```bash
npm run dev
```

Your application will be available at [http://localhost:3000](http://localhost:3000).

## Real-time Features Setup

To test Socket.IO and WebSocket functionality, set up a local test server:

### 1. Create Test Server

```bash
mkdir socket-test-server
cd socket-test-server
npm init -y
npm install express socket.io
```

### 2. Server Configuration

Create `server.js`:

```javascript
import express from "express";
import http from "http";
import { Server } from "socket.io";

const app = express();
const server = http.createServer(app);

const io = new Server(server, {
  cors: {
    origin: "*",
    methods: ["GET", "POST"],
    credentials: true,
  },
  transports: ["websocket", "polling"],
  pingInterval: 25000,
  pingTimeout: 5000,
});

io.on("connection", (socket) => {
  console.log("âœ… Client connected:", socket.id);
  
  socket.emit("welcome", {
    message: "Hello from Socket.IO server!",
    id: socket.id,
    timestamp: new Date().toISOString(),
  });

  socket.on("message", (data) => {
    console.log("ðŸ“© Message received:", data);
    socket.emit("message", { 
      echo: data, 
      receivedAt: new Date().toISOString() 
    });
  });

  socket.on("disconnect", (reason) => {
    console.log("âŒ Client disconnected:", socket.id, "Reason:", reason);
  });
});

app.get("/", (req, res) => {
  res.send("Socket.IO Test Server is running âœ…");
});

const PORT = process.env.PORT || 4000;
server.listen(PORT, () => {
  console.log(`ðŸš€ Socket.IO server running on http://localhost:${PORT}`);

```

## Common Issues

### Port Already in Use

If port 3000 is already in use:

```bash
npm run dev -- -p 3001
```

### Node Version Issues

Use Node Version Manager (nvm) to switch Node versions:

```bash
nvm install 18
nvm use 18
```

### Package Installation Errors

Clear npm cache and reinstall:

```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```
